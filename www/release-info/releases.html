Release cycle

The numbers below are in weeks for minor releases.

  • 0 - release candidate of version X-1
  • 0 - start development of version X
  • 2 - release of version X-1
  • 3 - last minor fix of version X-1
  • 6 - freezing API of major new features in version X
  • 8 - feature freeze for new features in version X
  • 10 - feature freeze for user-requested features in version X
  • 10 - release candidate of version X
  • 10 - start development of version X+1
  • 12 - release of version X
  • 13 - last minor fix of version X

Each minor version usually takes twelve weeks of development with releases every ten weeks.
